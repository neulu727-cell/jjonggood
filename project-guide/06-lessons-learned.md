# 개발 이력 & 교훈

## 실수에서 배운 것

### 1. 변수 참조 확인 없이 코드 삭제
**상황**: `renderTimelineList`에서 과거 날짜 필터 로직을 제거하면서 `isPast` 변수 선언도 함께 삭제.
하지만 `isPast`는 아래에서 상태 뱃지 표시와 빈 슬롯 숨김에 사용 중이었음.

**결과**: `ReferenceError` → 타임라인 전체 크래시

**교훈**: 변수를 삭제하기 전에 **파일 전체에서 해당 변수의 모든 참조를 확인**할 것.
```javascript
// 삭제하려는 줄
const isPast = selectedDate < fmtDate(new Date());
// → 먼저 "isPast" 검색해서 다른 곳에서 쓰는지 확인
```

---

### 2. z-index 미고려로 시트가 뒤에 숨음
**상황**: 고객 상세(z:100) 위에 예약 상세를 열었는데, 예약 상세도 z:100이라 DOM 순서에 따라 뒤에 깔림.

**교훈**: 시트가 겹칠 수 있는 구조면 **z-index를 명시적으로 계층화**할 것.
```
기본 시트: z-index: 100
위에 쌓이는 시트: z-index: 110
팝업: z-index: 200
토스트: z-index: 300
```

---

### 3. 새 시트 열 때 기존 시트를 닫아버림
**상황**: 예약 상세를 열면서 `closeSheet('customerDetailSheet')`를 호출.
예약 상세를 닫으면 고객 상세도 이미 닫혀있어서 사라짐.

**교훈**: 시트는 **독립적으로 열고 닫아야** 한다. 새 시트를 열 때 다른 시트를 닫지 말 것.

---

### 4. siblings 쿼리에 필요한 필드 누락
**상황**: 펫 스위처에서 강아지별 메모를 보여주려 했으나, `get_siblings` 쿼리의 SELECT에 `memo` 필드가 없어서 모든 강아지가 같은 메모를 표시.

**교훈**: 프론트에서 새로운 필드를 쓸 때 **백엔드 쿼리에 해당 필드가 포함되는지** 확인할 것.

---

### 5. 캘린더와 타임라인의 데이터 불일치
**상황**: 캘린더 뱃지는 모든 예약을 표시하는데, 타임라인은 과거 날짜에서 완료된 것만 표시.
→ "캘린더에 예약이 보이는데 타임라인에는 없다"

**교훈**: **같은 데이터를 보여주는 두 뷰는 동일한 필터 로직**을 사용해야 한다.

## UI/UX 피드백 히스토리

### "짜치는" UI → 정보 밀도 높이기
- 빈 공간이 많으면 "성의없는" 느낌
- 관련 정보 한 줄로 합치기 (몸무게+반려동물, 금액+결제방법)
- PC에서 무조건 2열 레이아웃

### "성의없는" textarea → 포커스 스타일링
```css
textarea#quickMemo:focus {
    border-color: var(--primary);
    box-shadow: 0 0 0 3px rgba(79, 70, 229, 0.1);
}
```
기본 textarea에 border만 있으면 밋밋함. 포커스 시 색상 변화와 글로우 효과 필수.

### "메모 저장" → "메모 추가"
- 저장 = 덮어쓰기 느낌 → 기존 메모 날릴까 불안
- 추가 = 기존 것에 더하는 느낌 → 안전하고 직관적

### 상태변경 섹션 불필요
- 별도 "상태변경" 섹션은 화면만 차지
- 액션 버튼에 직접 기능 통합 (미용완료, 예약취소, 노쇼)

### 결제방법은 완료 시점에
- 예약 등록 시 결제방법 → 무의미 (아직 미용 안 했으니까)
- 완료 버튼 클릭 시 팝업으로 카드/현금 선택

### 통합 상세 페이지 리디자인 교훈 (2026.03.13)

**문제**: 예약상세/고객상세를 통합했지만, 통계를 큰 박스 3개로 배치하고 label:value 테이블로 정보를 나열 → PC에서 스크롤이 필요할 만큼 공간 낭비.

**핵심 깨달음: 박스가 아니라 텍스트가 일한다**

| 잘못한 것 | 고친 것 | 원칙 |
|----------|--------|------|
| 통계 3박스 (방문/매출/평균) | `12회 방문 · 매출 850,000원` 인라인 | 박스로 위계 잡지 말고 **font-weight**로 |
| "평균 단가" 표시 | 제거 | 파생 지표는 넣지 않는다 |
| 현재 예약 label:value 4행 | 2줄 문장 + 좌측 액센트바 | 테이블 형식 대신 **자연어 밀도** |
| 정보수정 full-width 버튼 | 텍스트 링크 | 보조 액션은 **최소 공간** |
| 예약 이력 → 별도 모달 | 아코디언 (그 자리에서 펼침) | 컨텍스트 전환 최소화 |

**공간 절약 규칙**:
1. 숫자를 돋보이게 하고 싶으면 **bold + 색상**이지 큰 박스가 아니다
2. label:value 테이블은 5행 이상이면 반드시 인라인 문장으로 압축
3. 파생 지표(평균, 비율 등)는 핵심이 아니면 제거
4. full-width 버튼은 "주 액션" 1개에만 허용. 보조 액션은 텍스트 링크
5. 구분선/테두리 대신 좌측 액센트바(3px) + 들여쓰기로 영역 구분

## 성능 최적화 이력

### N+1 쿼리 제거
고객 목록에서 각 고객마다 방문 수를 별도 쿼리하던 것 → JOIN으로 통합.
**체감 속도 대폭 개선.**

### DB 커넥션 관리
- keepalive로 죽은 연결 자동 감지
- 앱 시작 시 미리 연결 (첫 요청 지연 제거)
- 커넥션 풀링 적용

### KST 시간대 문제
UTC로 저장되어 시간이 맞지 않던 문제 → KST(한국시간)로 통일.

## 기능 진화 순서

```
1. 기본 예약 CRUD
2. 캘린더 + 타임라인 뷰
3. 멀티펫 지원 (phone+pet_name UNIQUE)
4. ADB Bridge (전화 수신 → 자동 고객 매칭)
5. PC 3컬럼 레이아웃
6. 고객 등록 폼 간소화 (7필드 → 4필드)

8. 결제방법 완료 시점 이동
9. 예약 상세 2열 레이아웃 + 정보 압축
10. 강아지별 메모 인라인 편집
11. 고객 상세 2열 와이드 + 예약메모 표시
12. notes→memo DB 마이그레이션
13. 예약상세+고객상세 → 통합 상세 페이지 (unifiedDetailSheet)
14. 통합 상세 컴팩트 재설계 (박스→인라인, label:value→문장, 아코디언 이력)
```

## 개발 원칙 요약

1. **최소 필드**: 등록 시 4개 이하. 나머지는 수정 폼에서.
2. **빈틈 없이**: 화면에 빈 공간 있으면 정보를 합치거나 레이아웃 조정.
3. **시점에 맞게**: 정보 입력은 해당 시점에. 미리 받지 않는다.
4. **추가 방식**: 메모는 덮어쓰기가 아닌 추가.
5. **통합 시트**: 예약+고객 상세는 하나의 통합 페이지. 이력은 아코디언.
6. **강아지 중심**: 보호자가 아닌 강아지가 핵심 단위.
7. **일관된 뷰**: 같은 데이터 = 같은 필터.
8. **버튼 우선**: 드롭다운보다 버튼 그리드.
9. **타이포그래피 위계**: 박스/카드가 아닌 굵기·크기·색상으로 정보 위계를 잡는다.
10. **파생 지표 제거**: 핵심이 아닌 평균·비율 등은 표시하지 않는다.
